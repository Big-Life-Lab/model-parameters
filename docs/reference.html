Model Parameter Files

5 Reference

5.1 variables

Contains all the variables needed for algorithm development and implementation

5.1.1 Columns

5.1.1.1 variable

The name of the variable

Type: string

5.1.1.2 role

The different processes that the variable was part of during the algorithm development

Type: string

5.1.1.3 variableType

The statistical type of the variable

Type: category

Category Value Category Description
Categorical Categorical type
Continuous Continuous type

5.1.1.4 databaseStart

The databases that the variable can be harmonized from

Type: string

5.1.1.5 variableStart

The names of the variable for each starting database

Type: string

5.1.1.6 units

The units for the variable

Type: string

5.1.1.7 label

A short description of the variable

Type: string

5.1.1.8 labelLong

A long description of the variable

Type: string

5.1.1.9 description

Should contain any issues that need to be taken into account before using this variable

Type: string

5.2 variable-details

Contains the details for the variables defined in a variables sheet

5.2.1 Columns

5.2.1.1 variable

The name of the variable

Type: string

5.2.1.2 dummyVariable

The name of the dummy variable. Not valid if the variable is continuous.

Type: string

5.2.1.3 typeEnd

The type of the variable

Type: category

Category Value Category Description
cat Categorical type
cont Continuous type

5.2.1.4 databaseStart

The starting databases whose harmonization details this row contains

Type: string

5.2.1.5 variableStart

The names of the variable for each starting database

Type: string

5.2.1.6 typeStart

The type of the starting variables

Type: category

Category Value Category Description
cat Categorical type
cont Continuous type
N/A Used mainly for derived variables when there are many start variables

5.2.1.7 recEnd

The value that the variable will be harmonized to. Can be a number or an interval.

Type: string

5.2.1.8 catLabel

The description for the category.

Type: string

5.2.1.9 catLabelLong

A longer description for the category.

Type: string

5.2.1.10 numValidCat

The number of valid categories for the variables.

Type: number

5.2.1.11 units

The units for the variable. Only for continuous variables.

Type: string

5.2.1.12 recStart

The value of the starting variable to harmonize from. Can be a single value or an interval.

Type: string

5.2.1.13 catStartLabel

The description for the category of the start variable.

Type: string

5.2.1.14 variableStartShortLabel

A short description of the starting variables

Type: string

5.2.1.15 variableStartLabel

A long description of the starting variables

Type: string

5.2.1.16 notes

Any issues/problems that users of this variable should consider. Can also include any problems encountered during harmonization.

Type: string

5.3 lookup

Contains the score range that each bin belongs to

5.3.1 Columns

5.3.1.1 catValue

The bin number

Type: number

5.3.1.2 range

The range if score values for this bin. Should use the mathematical range notation.

Type: string

5.4 descriptive

Contains descriptive information, for example mean, median etc. for variables

5.4.1 Columns

5.4.1.1 variable

The variable whose descriptive statistics the row contains

Type: string

5.4.1.2 catValue

The value of the category whose descriptive statistics the row contains. N/A for continuous variables.

Type: number

5.4.1.3 n

The number of individuals used to calculate the row’s statistics

Type: number

5.4.1.4 proportion

The proportion of all the individuals in the study used to calculate this row’s statistics

Type: number

5.4.1.5 median

The median statistic

Type: number

5.5 model-export

Contains the list of all files that are part of the export for an algorithm

5.5.1 Columns

5.5.1.1 fileType

The type of file

Type: category

Category Value Category Description
variables A variables file
variable-details A variable details file
descriptive A descriptive lookup file
descriptive-bins A descriptive bins file
model-export A model export file
model-steps A model steps
dummy A dummy file
center A centering file
rcs An RCS file
interaction An interaction file
fine-and-gray A fine and gray file
cox A cox file
survival_bins File with survival data for the different bins
lookup Contains the range of score values for each biin in a survival algorithm
validate Contains the validation rules for variables in the algorithm

5.5.1.2 filePath

The path to the file relative to the model export file

Type: string

5.6 model-steps

Contains the steps to score an algorithm

5.6.1 Columns

5.6.1.1 step

The type of step to perform

Type: category

Category Value Category Description
dummy Step used to create dummy variables
center Step used to create centered variables
rcs Steps used to create spline terms
interaction Step used to create interaction terms
fine-and-gray Step used to calculate the outcome of a fine and gray model
cox Step used to calculate the outcome of a cox proportional hazards model

5.6.1.2 fileType

The type of file referenced in the file path column

Type: category

Category Value Category Description
N/A Missing since certain steps don’t need to specify the file type
beta-coefficients A beta coefficients file for a fine and gray model or a cox model
baseline-hazards The baseline hazards for a fine and gray model or a cox model

5.6.1.3 filePath

The path to the file relative to the model location of the model steps file

Type: string

5.6.1.4 notes

Any notes for the future

Type: string

5.11 validate

Contains the validation rules for variables in the algorithm

5.11.1 Columns

5.11.1.1 variable

The name of the variable to validate

Type: string

5.11.1.2 rule

The type of validation rule to apply to a variable

Type: category

Category Value Category Description
type Validate the data type of the variable
range Validate that the variable is within a range of values. Meant for continuous variables.
allowed Validate that the variable contains a value from a set of valid values. Meant for categorical variables.
nullable Whether missing values are allowed for a variable

5.11.1.3 value

The value to use when applying the validation

Type: string

5.11.1.4 error_handle

How to handle failed validations

Type: category

Category Value Category Description
error Failed validations for the variable should throw an error, stopping the scoring process
warning Failed validations for the variable should log a warning and continue the scoring process
truncate Failed validations for the variable should log a warning, truncate the failed variable value, and continue scoring process

5.11.1.5 error_replace

Value to replace variables that fail validations whose errorHandle value is warning

Type: string

5.11.1.6 location

Which step in the scoring process the validation should be used

Type: string
